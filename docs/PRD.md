# E.O.M PRD

## 1. 제품명

E.O.M (Echo of Movement)

## 2. 한 줄 정의

인스타그램에 흩어진 스트릿 댄서들의 포트폴리오, 캐스팅, 행사, 네트워킹 정보를 하나의 아지트에 모으는 댄서 커뮤니티 플랫폼.

## 3. 배경과 문제

스트릿 댄서들은 주로 인스타그램, DM, 오픈채팅, 지인 추천을 통해 활동 기회를 찾는다. 이 방식은 접근성이 낮고 정보가 흩어져 있어, 새로운 댄서가 일을 구하거나 팀원을 찾거나 행사를 발견하기 어렵다.

E.O.M은 댄서들이 이미 쓰는 인스타그램 문화와 연결되면서도, 필요한 활동 정보를 목적별로 모아 보여주는 공간을 만든다. 댄서 플랫폼 특성상 이미지보다 움직임을 보여주는 영상, 릴스, 게시물 URL 기반 콘텐츠가 핵심이다.

## 4. 목표 사용자

- 공연, 배틀, 연습 영상을 기록하고 싶은 댄서
- 강사, 백업댄서, 크루원을 구하는 사람
- 배틀, 워크숍, 스트릿 행사를 홍보하는 사람
- 함께 연습할 파트너나 팀원을 찾는 댄서
- 커뮤니티 게시글과 사용자를 관리하는 관리자

## 5. 핵심 가치

- 나를 보여주기 쉽다.
- 기회를 찾기 쉽다.
- 씬의 행사와 모집 정보를 한곳에서 볼 수 있다.
- 인스타그램 활동과 자연스럽게 이어진다.
- 로그인 후 댄서들의 아지트 같은 분위기를 제공한다.
- 공개 index의 SHOW/CAST/HYPE/LINK CTA는 비로그인 상태에서는 로그인으로, 로그인 상태에서는 각 보드 목록으로 바로 이동한다.

## 6. MVP 필수 기능

### 인증

- 사용자 로그인
- 관리자 로그인
- 로그인 상태 유지는 기본 체크 상태로 제공하며, remember-me 쿠키로 브라우저를 오가도 인증 상태를 유지
- `/`, `/login`, 정적 리소스, H2 콘솔은 공개 접근이고 그 외 화면은 로그인 필요
- `/admin/**`은 ADMIN 권한 필요
- 로그아웃은 공통 헤더가 아니라 마이페이지 내부에서 수행
- 데모 계정 제공

### 대시보드

- 로그인 후 첫 화면
- 최근글, 인기글, 다가오는 행사, 추천 피드 영역
- Today Pick, Popular, Recent, Tags, Activity, Events, Dancers 영역으로 구성
- Today Pick의 `VIEW POST`, Popular/Recent 게시글 제목과 미리보기는 내부 게시글 상세 `/posts/{id}`로 이동
- Recent 영역은 기본값을 SHOW로 두고 `SHOW`, `CAST`, `HYPE`, `LINK` 탭을 페이지 전체 새로고침 없이 전환
- Recent 탭은 선택된 보드의 더미데이터 최신 글 10개만 미리보기로 노출
- Recent의 `ALL` 및 헤더의 SHOW/CAST/HYPE/LINK는 보드별 목록 `/boards/{board}`로 이동
- Header 검색은 `/posts?q={query}` 통합 검색으로 이동하며, Tags의 `ALL`은 `/posts` 검색 화면, 개별 Tags는 호환용 태그 검색 결과 `/posts?tag={tag}`로 이동
- Activity의 `ALL`은 별도 Activity 화면이 아니라 전체 게시글 목록 `/boards/all`로 이동
- Events는 HYPE 보드의 관리자 승인 행사 필터(`/boards/HYPE?officialEvents=true`)로 이동
- Dancers는 장르별 댄서 탐색 페이지 `/dancers`로 이동
- 추천 댄서의 Follow는 사용자의 인스타그램 프로필을 새 탭으로 열기
- 인스타그램, 유튜브 등 외부 미디어 링크는 게시글 상세에서 새 탭 링크로만 다루고, 대시보드와 목록 썸네일 자체는 내부 게시글 미리보기 역할만 수행

### 보드

- SHOW: 포트폴리오, 공연 기록, 연습 영상, 안무 쉐어 등 나의 활동을 기록하고 보여주는 공간
- CAST: 강사, 백업댄서, 팀원, 오디션 모집처럼 함께 움직일 사람과 기회를 찾는 공간
- HYPE: 배틀, 워크숍, 쇼케이스, 파티와 같은 현장의 열기와 일정을 공유하는 공간
- LINK: 연습 파트너, 촬영 메이트, 공간 공유, 피드백 모임처럼 댄서들이 서로 연결되는 공간

전체 목록은 `/boards/all`, 각 보드는 `/boards/SHOW`, `/boards/CAST`, `/boards/HYPE`, `/boards/LINK` 목록과 `/posts/{id}` 상세 흐름을 가진다. 목록은 기본 최신순이며 `sort=latest|views|comments|likes`로 최신순, 조회순, 댓글순, 좋아요순 정렬을 지원한다. 검색 화면 `/posts`는 빈 검색어에서 전체 목록으로 흐르지 않고 검색 안내와 추천 태그를 보여주며, 검색어가 있을 때 `tags`, `title`, `content`, `author.displayName`, `author.crewName`을 통합 검색한다. `#`로 시작하는 헤더 검색어는 `#`를 제거해 같은 통합 검색으로 처리하고, `/posts?tag={tag}`는 태그 클릭 호환 경로로 유지한다. 작성 흐름은 후속 범위로 둔다.

### HYPE 관리자 승인 행사

- 공식 행사, 배틀, 공연은 HYPE 보드 안에서 `관리자 승인 행사` 필터로 탐색
- `/boards/HYPE?officialEvents=true`는 `adminApprovedEvent=true`인 HYPE 글만 보여주며 기존 최신순/조회순/댓글순/좋아요순 정렬을 유지
- `/events`는 기존 링크 호환을 위해 관리자 승인 행사 필터가 켜진 HYPE 보드로 리다이렉트
- 목록에는 `eventDate`, `deadline`, `location`, `boardType`, `mediaType`, `thumbnailUrl`, `title`, `content` preview, `author.displayName`을 노출
- 외부 미디어 링크는 상세 화면에서만 새 탭으로 열고, 목록에서는 썸네일과 내부 상세 링크만 제공
- 직접 embed, 직접 영상 업로드, 새 이미지 원본 추가는 MVP 범위에서 제외

### Dancers

- `/dancers`는 댄서를 빠르게 비교하는 탐색 화면으로, `AppUser` 중 USER 역할 사용자를 댄서 카드로 렌더링
- 히어로는 기존 스트릿 디렉터리 톤을 유지하고, 본문은 인재 카드 검토처럼 이름, 아이디, 주 장르, 크루, 연락 가능 여부, 소개를 정리
- 모든 댄서 사진 미리보기는 동일한 카드 영역과 동일한 비율로 노출
- 장르 필터는 `All`, `Hip-hop`, `House`, `Krump`, `Popping`, `Locking`, `Breaking`, `Waacking`, `Voguing`, `Dancehall` 버튼을 제공
- 장르 버튼은 다중 선택 가능하며 `/dancers?genres={genre}` 쿼리를 여러 개 받아 선택 장르 중 하나라도 매칭되는 댄서를 노출
- `All`은 선택된 필터를 초기화하고 전체 댄서를 노출
- 각 댄서 카드는 인스타그램 링크를 새 탭으로 열고, 프로필 보기는 기존 `/dancers/{id}` 작성자/댄서 프로필 이동 흐름을 유지

### 마이페이지

- URL은 `/my-page`를 기본으로 사용하고 `/me`도 같은 화면으로 연결
- displayName, crewName, bio, instagramUrl, profileImageUrl, headerImageUrl, 주 장르를 보여주는 프로필 히어로
- 공통 헤더는 일반 화면에서 `My Page` 버튼을 보여주고, 마이페이지에서는 같은 위치를 붉은 `Logout` 버튼으로 전환
- admin과 일반 사용자 모두 본인 마이페이지에 접근 가능
- 내가 쓴 전체 게시글을 포트폴리오 후보로 취급
- 사용자가 원하는 게시글을 포트폴리오 탭에 선택하고, 선택된 포트폴리오 중 최대 3개를 상단 고정
- 탭 순서는 Portfolio, Posts, Events, Likes, Comments
- Portfolio 탭은 선택한 포트폴리오만 보여주고, Posts 탭은 내가 쓴 전체 게시글을 카드형으로 보여주며 별 버튼으로 포트폴리오 포함 여부를 바꿈
- Events 탭은 사용자가 직접 기입한 참여 행사 전체 목록과 입력 폼을 카드형으로 제공
- Likes 탭은 좋아요한 게시글을 목록형으로 제공
- Comments 탭은 내가 작성한 댓글을 목록형으로 제공
- Activity는 내가 작성한 게시글과 댓글 기반 최근 활동을 자동 표시
- 사이드바 Joined Events는 최신 참여 행사 일부만 보여주는 프리뷰
- 프로필 편집은 헤더 이미지 URL, 프로필 이미지 URL, 인스타그램 URL, 이름, 크루, 주 장르, 소개를 수정
- 계정 편집은 아이디 변경과 새 비밀번호 변경을 지원하며 현재 비밀번호 확인 후 저장하고, 저장 후 재로그인을 요구
- 팔로워/팔로잉 수는 MVP 마이페이지에서 제공하지 않음

### 인스타그램 연동

- 프로필에 인스타그램 링크 첨부
- 게시글에 인스타그램 릴스/게시물 URL, 유튜브 URL, 외부 영상 URL 첨부
- MVP에서는 직접 영상 파일 업로드를 지원하지 않음
- 목록 화면은 `thumbnailUrl`을 카드 이미지로 사용
- 상세 화면은 `thumbnailUrl` 중심으로 미디어를 보여주고 `mediaType`이 IMAGE가 아니면 외부 미디어임을 표시합니다.
- `mediaUrl`과 `instagramUrl`은 상세 화면에서 새 탭 링크로 열며, 실제 embed 또는 직접 영상 업로드는 구현하지 않습니다.
- 인스타그램 URL은 `https://www.instagram.com/hazyala?igsh=ZW1maGFzNHQzdzEx&utm_source=qr` 계정과 해당 계정 내 확인 가능한 실제 릴스/게시물 URL을 기준으로 함
- 정확한 릴스/게시물 URL을 확인한 경우 해당 URL을 사용하고, 확인 전에는 위 프로필 URL을 fallback으로 사용한 뒤 실제 릴스/게시물 URL로 교체 예정

### 관리자

- 관리자 전용 페이지
- 사용자 목록 조회
- 게시글 목록 조회
- 게시글 삭제와 사용자 관리 액션은 후속 범위

### 테마

- 기본값 다크 모드
- 라이트/다크 토글
- 선택값 localStorage 저장
- 색상은 CSS 변수로 관리

## 7. 주요 사용자 시나리오

### 댄서가 자신을 어필하는 흐름

1. 로그인한다.
2. SHOW 보드로 이동한다.
3. 공연/연습 기록과 인스타그램 URL을 첨부해 글을 작성한다.
4. 다른 사용자가 글과 인스타그램 링크를 보고 연락할 수 있다.

### 팀원을 구하는 흐름

1. 로그인한다.
2. CAST 보드로 이동한다.
3. 모집 조건, 일정, 장소, 인스타그램 연락 링크를 등록한다.
4. 관심 있는 사용자가 글을 확인하고 연락한다.

### 행사를 홍보하는 흐름

1. 관리자 또는 사용자가 HYPE 보드에 행사 글을 작성한다.
2. 대시보드와 HYPE 보드에서 노출된다.
3. 인스타그램 게시물 또는 링크로 상세 홍보 페이지에 연결된다.

### 관리자가 커뮤니티를 관리하는 흐름

1. 관리자 계정으로 로그인한다.
2. 관리자 페이지로 이동한다.
3. 사용자와 게시글 목록을 확인한다.
4. 삭제/제재 같은 관리 액션은 후속 범위에서 연결한다.

## 8. 성공 기준

- 로컬에서 전체 주요 화면을 이동할 수 있다.
- 사용자/관리자 로그인 흐름이 동작한다.
- 관리자 페이지가 권한으로 보호된다.
- 4개 보드의 목록/상세 흐름이 동작한다.
- 헤더 검색, 태그 검색, 빈 검색어 안내가 동작한다.
- HYPE 보드의 `관리자 승인 행사` 필터에서 공식 행사성 글을 탐색할 수 있다.
- `/my-page`에서 프로필/계정 수정, 포트폴리오 선택, 참여 행사 기록을 관리할 수 있다.
- 인스타그램 링크 또는 게시물 URL이 프로필/게시글에 표시된다.
- Render 무료 티어에 배포 가능한 설정을 갖춘다.
- 기본 다크 모드와 라이트 모드가 모두 동작한다.

## 9. 비범위

3일 MVP에서는 다음을 제외한다.

- 인스타그램 Graph API 공식 연동
- 실시간 채팅
- 결제/정산
- 고급 추천 알고리즘
- 대용량 이미지 업로드 서버
- 직접 영상 파일 업로드
- 직접 이미지 파일 업로드
- 게시글 작성/삭제 폼
- 외부 미디어 직접 embed
- 복잡한 관리자 권한 체계
